# 📈 Архітектурний та FinOps аналіз AI-провайдерів (Multi-LLM Стратегія)

Для забезпечення відмовостійкості, економії бюджету та уникнення Vendor Lock-in, інфраструктура `JobMatch` побудована за принципом **Multi-Provider Routing**. `AgentGateway` дозволяє динамічно перемикати моделі без зміни коду бекенду.

## 🏗️ Стратегія маршрутизації (Tiered Routing)

1. **Tier 1 (Економ/Швидкість)**: Усі рутинні операції (пошук, очищення даних) за замовчуванням ідуть на **Gemini 3 Flash** / **Flash-Lite**. Це економить до 85% бюджету хакатону.
2. **Tier 2 (Якість/Логіка)**: Складні завдання (генерація супровідних листів, адаптація під вакансію) маршрутизуються на **Claude Haiku 4.5** або **GPT-5.4**.
3. **Tier 3 (Валідація/Суддя)**: Запуск нашої автоматичної системи оцінки якості (Eval-suite / LLM-as-a-Judge) виконується на **Gemini 3.1 Pro**.
